## 👥 PARTNER ONBOARDING

### **📱 STEP 1: SITE ASSESSMENT**
```bash
□ Greenhouse Location Survey:
  - WiFi signal strength test
  - Power outlet availability
  - Environmental conditions
  - Distance from router
  - Interference sources

□ Infrastructure Requirements:
  - Stable WiFi (2.4GHz preferred)
  - Power outlet near ESP32
  - Router access for configuration
  - Smartphone for dashboard access
```

### **🔧 STEP 2: HARDWARE INSTALLATION**
```bash
□ ESP32 Mounting:
  - Protected from direct rain
  - Good ventilation
  - Near sensors and relays
  - Accessible for maintenance

□ Sensor Placement:
  - DHT22: 1.5m height, shaded area
  - Soil Moisture: 10-15cm depth
  - Light Sensor: Facing upward, no obstruction
  - All sensors connected securely

□ Relay Installation:
  - Pump relay: In series with pump power
  - Fan relay: In series with fan power
  - LED relay: In series with LED power
  - All relays accessible but protected
```

### **⚡ STEP 3: POWER & NETWORK SETUP**
```bash
□ Power Connection:
  - 5V 2A power adapter minimum
  - Surge protector recommended
  - Backup power source (optional)
  - Cable management and protection

□ WiFi Configuration:
  - SSID: Partner's network
  - Password: Strong and stable
  - Signal strength: > -70dBm at ESP32
  - Static IP recommended (optional)
```

## 🚀 DEPLOYMENT DAY

### **📋 INSTALLATION PROCEDURE**
```bash
1. SITE PREPARATION (30 minutes):
   - Mount ESP32 in selected location
   - Install sensors in correct positions
   - Connect relays to control devices
   - Organize and protect cables

2. POWER ON & CONFIGURATION (15 minutes):
   - Connect power adapter
   - Monitor Serial for connection status
   - Verify WiFi connection
   - Confirm Firebase connection

3. SYSTEM TESTING (30 minutes):
   - Test all sensor readings
   - Test all manual controls
   - Verify auto-mode functionality
   - Test fail-safe mechanisms

4. USER VERIFICATION (15 minutes):
   - Partner tests dashboard
   - Partner tests controls
   - Partner confirms understanding
   - Document any issues

5. FINAL HANDOVER (15 minutes):
   - Provide user manual
   - Share support contacts
   - Set up monitoring
   - Schedule follow-up
```

## 🚨 EMERGENCY PROCEDURES

### **⚡ IMMEDIATE RESPONSE**
```bash
□ SYSTEM DOWN:
  1. Check power supply
  2. Check WiFi connection
  3. Restart ESP32
  4. Contact support if needed

□ PUMP STUCK ON:
  1. Use dashboard emergency OFF
  2. Manual power cutoff if needed
  3. Check relay connections
  4. Contact support immediately

□ SENSORS NOT WORKING:
  1. Check sensor connections
  2. Verify sensor placement
  3. Replace faulty sensors
  4. Recalibrate system
```
